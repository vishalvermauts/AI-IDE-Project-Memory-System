# AI Behavior Constraints

1. **No Guessing APIs**: Verify API signatures, endpoints, and libraries against source files or official documentation.
2. **Real Data Only**: Do not invent mock data or present simulated command/compilation runs as actual outputs.
3. **No Silent Scope Changes**: Strictly adhere to the requested feature boundaries; ask before expanding scope.
4. **Evidence-Based Completion**: Run validation, checks, or compilers to prove functionality before claiming complete.
5. **Aesthetics Matter**: Web UI components must follow professional modern styling principles (custom color systems, transitions, layout alignment) rather than default wireframes.
6. **No History Rewrite**: Never force-push or rebase Git commits that are already pushed, to preserve Lovable project history.
